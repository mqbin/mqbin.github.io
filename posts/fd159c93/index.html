06Local Minima and Saddle Point

发表于2021-09-10|更新于2021-09-15
|字数总计:2.9k|阅读时长:10分钟|阅读量:

When gradient is small

Critical Point

训练失败的原因是什么？

现在我们要讲的是Optimization的部分，所以我们要讲的东西基本上跟Overfitting关系不大，现在只讨论再Optimization的时候，怎么把gradient descent做的更好，那么为什么Optimization会失效呢？

根据这个图来看就是说，随着我们不断的train，loss也在不断的下降，但是我们对这个loss仍然不满意，就像我刚才说的，我们可以把deep的network和linear的model，或者比较shallow 的network比较，发现说它没有做的更好，所以我们认为deep network没有发挥出应有的力量，所以Optimization是有问题的。

但有时候我们会发现，一开始咱们的model就train不起来，不管我们怎么调整update参数，loss就是掉不下去！为什么？

之前我们说过了local minima的问题，在local minima处斜率为0，当参数对loss微分为0的时候，gradient descent就不能继续下去了。

其实除了local minima会为0，还有saddle point的问题，所谓的saddle point其实就是gradient是0，但不是local minima，像local minima和saddle point这一类的点我们统称为critical point。

其实local minima在实际中基本不会见到，如果把卡在了local minima这个事情写在paper上会让别人感觉非常业余。

你可以说你的loss,没有办法再下降,也许是因為卡在了critical point,但你不能说是卡在local minima,因為saddle point也是微分為零的点

那如果gradient真的接近0了，卡在了某一个critical point我们有没有办法知道，到底是local minima还是saddle point呢？

有的。

为什么我们想知道是卡在local minima还是saddle point呢？

  • 因为如果是卡在local minima，那就不能回头了，四周都比较高，已经出不去了
  • 但是如果是saddle point的话就没有这个问题，如果你今天卡在了saddle point的话，saddle point旁边还是有路可以走的，有路可以让loss走向更低。

所以鉴别今天我们走到，critical point的时候，到底是local minima还是saddle point？

数学知识

针对loss function的形状，我们怎么能知道呢？network很复杂，在算出来loss function，显然也十分复杂

泰勒级数近似

如果给定某一组参数，比如说蓝色的θ′\theta'θ′，在这个θ′\theta'θ′附近的loss function是有办法被写出来的

这个完整L(θ)L(θ)L(θ)的样子是写不出来的，但是它在θ′\theta'θ′附近，你可以用这个式子来表示它，这个式子是Tayler Series Appoximation泰勒级数展开。

  • 展开式的第一项L(θ′)L(\theta')L(θ′)就说明当θ\thetaθ和θ′\theta'θ′很接近的时候，L(θ′)L(\theta')L(θ′)应该跟L(θ′)L(\theta')L(θ′)还蛮靠近的。

  • 第二项(θ−θ′)Tg(θ-θ')^Tg(θ−θ′)Tg，其中g是一个vector，就是我们的gradient

  • 第三项有一个Hession，这边有一个H

    这个H叫做Hessian，他是一个矩阵，这个第三项是，(θ−θ′)TH(θ−θ′)(θ-θ')^TH(θ-θ')(θ−θ′)TH(θ−θ′)，H里面存放的是对L的二次微分。

所以L(θ)L(\theta)L(θ)这个式子跟两个东西有关系，第一个跟gradient有关系，跟hessian有关系，gradient就是一次微分，hessian就是里面有二次微分的项目。

Hession

当我们今天走到了critical point，意味着gradient为0了，也就是绿色圈这一项全部不见了

critical point处g全为0，也就是说这种情况下我们要看它是哪一种critical point就是要看Hessian，这一项的正负

我们将(θ−θ′)(θ-θ')(θ−θ′)用vvv这个向量来表示

  • 如果对于任何可能存在的v，vTHvv^THvvTHv都是大于0的，这是一个正定二次型。所以L(θ)>L(θ′)L(θ)>L(θ')L(θ)>L(θ′)，在θ′\theta'θ′处是个最低点，local minima
  • 如果对于任何可能存在的v，vTHvv^THvvTHv都是小于0的，这是一个负定二次型。所以L(θ)<L(θ′)L(θ)<L(θ')L(θ)<L(θ′)，代表说L(θ′)L(θ')L(θ′)是附近最高的一个点，所以他是local maxima
  • 第三个可能是**vTHvv^THvvTHv,有时候大於零 有时候小於零**，代表有时候L(θ)>L(θ′)L(θ)>L(θ')L(θ)>L(θ′)，有时候L(θ)<L(θ′)L(θ)<L(θ')L(θ)<L(θ′)，这代表这是一个saddle point

但是这边说我们要带所有的vvv，去看vTHvv^THvvTHv是大于0还是小于0，我们不可能说是将所有的v一个个拿出来试，要靠线代中二次型的性质来看。

我们主要去看H的eigen value

  • 所有的eigen value都是正的，条件成立，vTHv>0v^THv > 0vTHv>0，也就是说是一个local minima。
  • 所有的eigenvalue都是负的，vTHv<0v^THv < 0vTHv<0 local maxima
  • 如果eigen value有正有负，那就是saddle point

对于saddle point

如果卡在了saddle point我们不需要害怕，因为saddle point还是可以继续train的，因为H它不只可以帮助我们判断现在是不是在一个saddle point，它还给我们指出了参数，我们可以update的方向，就之前我们参数update的时候，都是看gradient看g，但是我们走到某个地方g变为了0，看g没用了，如果这是一个saddle point我们可以继续看H

假设μ\muμ是H的eigenvector特征向量，λλλ是u的eigen value特征值。

如果我们把这边的vvv换成μ\muμ,我们把μ\muμ乘在H的左边，跟H的右边，也就是μHμ\mu H\muμHμ

HμH\muHμ会得到λμλ\muλμ，因為μ\muμ是一个eigen vector。H乘上eigen vector特征向量会得到特征向量λ eigen value乘上eigen vector即λμλ\muλμ

所以我们在这边得到uᵀuᵀuᵀ乘上λuλuλu,然后再整理一下,把uᵀuᵀuᵀ跟uuu乘起来,得到‖u‖²‖u‖²‖u‖²,所以得到λ‖u‖²λ‖u‖²λ‖u‖²

​ 那今天如果λ小於零,eigen value小於零的话,那λ‖u‖²就会小於零,因為‖u‖²一定是正的,所以eigen value是负的,那这一整项就会是负的,也就是u的transpose乘上H乘上u,它是负的,也就是红色这个框裡是负的

​ 所以这意思是说假设θ−θ′=μθ-θ'=\muθ−θ′=μ,那这一项(θ−θ′)TH(θ−θ′)(θ-θ')^TH(θ-θ')(θ−θ′)TH(θ−θ′)就是负的,也就是L(θ)<L(θ′)L(θ)<L(θ')L(θ)<L(θ′)

​ 也就是说假设θ−θ′=μθ-θ'=\muθ−θ′=μ,也就是,你在θ’的位置加上u,沿著u的方向做update得到θ,你就可以让loss变小

​ 因為根据这个式子,你只要θ减θ’等於u,loss就会变小,所以你今天只要让θ等於θ’加u,你就可以让loss变小,你只要沿著u,也就是eigen vector的方向,去更新你的参数 去改变你的参数,你就可以让loss变小了

​ 所以虽然在critical point没有gradient,如果我们今天是在一个saddle point,你也不一定要惊慌,你只要找出负的eigen value,再找出它对应的eigen vector,用这个eigen vector去加θ’,就可以找到一个新的点,这个点的loss比原来还要低

​ 如果你今天在training的时候,你的gradient你的训练停下来,你的gradient变成零,你的训练停下来,是因為saddle point的话,那似乎还有解

​ 但是当然实际上,在实际的implementation裡面,你几乎不会真的把Hessian算出来,这个要是二次微分,要计算这个矩阵的computation,需要的运算量非常非常的大,更遑论你还要把它的eigen value,跟 eigen vector找出来,所以在实作上,你几乎没有看到,有人用这一个方法来逃离saddle point

​ 等一下我们会讲其他,也有机会逃离saddle point的方法,他们的运算量都比要算这个H,还要小很多,那今天之所以我们把,这个saddle point跟 eigen vector,跟Hessian的eigen vector拿出来讲,是想要告诉你说,如果是卡在saddle point,也许没有那麼可怕,最糟的状况下你还有这一招,可以告诉你要往哪一个方向走.

Saddle Point 和 Local Minima

讲到这边有问题了，saddle point和local minima谁更常见？

从三维的空间来看,是没有路可以走的东西,在高维的空间中是有路可以走的,error surface会不会也一样呢

所以你在一维的空间中,一维的一个参数的error surface,你会觉得好像到处都是local minima,但是会不会在二维空间来看,它就只是一个saddle point呢,常常会有人画类似这样的图,告诉你说Deep Learning的训练,是非常的复杂的,如果我们移动某两个参数,error surface的变化非常的复杂,是这个样子的,那显然它有非常多的local minima,我的这边现在有一个local minima,但是会不会这个local minima,只是在二维的空间中,看起来是一个local minima,在更高维的空间中,它看起来就是saddle point,在二维的空间中,我们没有路可以走,那会不会在更高的维度上,因為更高的维度,我们没办法visualize它,我们没办法真的拿出来看,会不会在更高维的空间中,其实有路可以走的,那如果维度越高,是不是可以走的路就越多了呢,所以 今天我们在训练,一个network的时候,我们的参数往往动輒百万千万以上,所以我们的error surface,其实是在一个非常高的维度中,对不对,我们参数有多少,就代表我们的error surface的,维度有多少,参数是一千万 就代表error surface,它的维度是一千万,竟然维度这麼高,会不会其实,根本就有非常多的路可以走呢,那既然有非常多的路可以走,会不会其实local minima,根本就很少呢,

总而言之，local minima是不常见的问题，尤其是在神经网络这种高维的情况下。

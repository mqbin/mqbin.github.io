General Guidance : overfit

之前已经看过了作业一，其实之后好几个作业，它看起来基本上都是大同小异。

就是你会有一堆训练的资料,这些训练集裡面,会包含了xxx跟yyy的hat,x¹x¹x¹ 和跟它对应的y^¹ŷ¹y^​¹,x²x²x² 跟它对应的y^²ŷ²y^​²,一直到xⁿxⁿxⁿ 还有它对应的y^ⁿŷⁿy^​ⁿ

测试集,测试集就是你只有x没有y,其实在之后每一个作业,看起来都是非常类似的格式

训练集是用来训练我们的Model，训练Model有三个步骤：

  • 第一步，先写出一个有未知数的function，fθ(x)f_θ(x)fθ​(x)，这就代表了一个对于x的的函数，里面有若干unknown的parameters，这个x叫做feature
  • 第二步，定义一个损失函数loss，loss的作用是输入一组parameters，去评判这组参数的好坏
  • 第三步，解Optimization的problem，要去找一个θ\thetaθ，这个θ\thetaθ可以让loss的值越小越好，让loss值最小的那个θ\thetaθ把它叫做θ∗\theta^*θ∗

得到θ∗\theta^*θ∗将其带入模型，放入测试集，也就是我们把输出存下，上传到Kaggle

如何优化我们的训练结果

如图从上到下走，如果感觉Kaggle的结果不满意，要做的第一件事情就是，检查training data 的loss。

这里可能有些疑问，说我关系的是testing data 的loss，kaggle上面也是testing data的结果。

但是你要先检查你的training data，先看这个Model 在training data上有没有训练起来，如果在训练集上的loss就很大，其实就是模型没有训练好，那这是什么的原因呢？

主要有两方面的原因：

  1. Model bias
  2. Optimization Issue

Model bias

model bias，就是设置的模型太过于简单，没有办法去拟合一个比较好的Model，也就是我们想找一个function但是由于模型本身太过于简单，导致这个function不在我们可以搜寻的范围内。

怎么解决Model bias呢？

我们可以重新设计Model，给这个Model更大的弹性，举例来说，我们可以增加输入的features，比如预测，我们一开始只给了前一天的信息，但是我们发现信息是有周期性的，我们可以将一个周期的信息都给它，这样Model就有了比较大的弹性，我们也可以用Deep Learning，增加更多的弹性。

Optimization

我们知道，今天使用的optimization，在这门课里面，我们其实只会用到gradient descent，这种optimization的方法有很多问题。

举例来说，有可能梯度下降会卡住在local minima的位置，导致我们没办法再进行下去。

这也就造成了我们最后得到的θ∗\theta^*θ∗的loss是不够低的，在这个model里面，存在一个function的loss是够低的，但是gradient descent并没有给我们这个function。

这边问题来了，training data的loss不够低的时候,到底是model bias,还是optimization的问题呢

一个比较建议的判断方法，就是你可以透过比较不同的模型，来得之说，你的model现在还够不够大

举例：residual network

paper链接：http://arxiv.org/abs/1512.03385

这篇paper一开头就讲，我想测两个networks

  • 一个network有20层
  • 一个network有56层

​ 我们把它们测试在测试集上,这个横轴指的是training的过程,就是你参数update的过程,随著参数的update,当然你的loss会越来越低,但是结果20层的loss比较低,56层的loss还比较高

在之前认命普遍认为太深的network并不work，会overfitting，但是这个是overfitting吗？

这个不是overfitting！

检查训练集的结果我们发现在训练集上20层的network，跟56层的network比起来，20层的是比较低的，56层的是比较高的。

这代表的是56层的network，它的optimization没有做好！

这里一定不会是Model bias，因为56层肯定要比20层的network更加灵活，不存在说Model bias的问题，就算是56层保持20层的样子后面的层不进行操作，也不比20层的差。

所以对于56层的network，如果我们训练成功了，它也应该比20层的network在训练资料上更好，可以得到更低的loss，所以这不叫overfitting，只是optimization没有做好的问题。

如何判断是不是Optimization没有做好？

我们可以先跑一些比较简单的network，有时候可以不先使用deep learning的方法，比如说，Linear Model，比如说支持向量机，这些是比较简单的，比较好train的，比较不会有optimization失败的问题。

这样我们就心里有数了，简单的Model已经可以得到了什么样的loss

更深的network没有得到简单模型的loss，就说明优化有问题。

接下来还缺一个深的model,如果你发现你深的model,跟浅的model比起来,深的model明明弹性比较大,但loss却没有办法比浅的model压得更低,那就代表说你的optimization有问题,你的gradient descent不给力,那你要有一些其它的方法,来把optimization这件事情做得更好。

假设现在经过努力，已经可以让training data的loss变小了，那么接下来就可以来看，testing data loss也小，有比这个strong baseline还要小就结束了。

但是如果我们觉得loss还不够小呢，如果说是在training data上面的loss比较小，testing data上面的loss大，那么我们可能就真的遇到overfitting的问题了。

所以

想知道怎么降低testing data的loss，首先就是要看training data的loss是不是有问题，但发现training loss已经很小了，但是testing loss反而变大了，这中情况就有可能是overfitting。

Overfitting

那么为什么会有overfitting的状况呢？为什么有可能training的loss小，testing的loss大呢？这边我们举一个极端的例子来告诉你说，为什么会发生这样子的状况

这是我们的一个训练集，根据训练集找出了一个很极端的function，我们可以发现，这个function的问题就是我们将训练集上的每一个x和y都对应起来了，但是遇到非训练集外的数据就会随机一个value，在training data上面，它的loss是0，但是在testing data上面loss就会变得特别大，因为它其实什么都没有预测，只是random了一下。

这种极端例子的问题在一般的状况下，也有可能会发生。

举例来说，假设我们输入的feature叫做x，我们输出的level叫做y，那x和y都是一维的。

x和y之间的关系，是这个二次曲线，这个曲线我们刻意用虚线来表示，因为我们基本没有办法可以观察到真实的曲线，我们真的可以观察到的是我们的训练集，训练集可以想象成从这条曲线上面随机sample出来的几个点

今天的模型的能力很强，它的flexibility很大，他的弹性很大的话，只给这三个点它只知道在这三个点的情况，其他没有给出点的地方它会做freestyle。

在上图我们也可以看出来经过训练了的模型model自由度很大，它可以产生非常奇怪的曲线，导致训练集上的结果好（因为学习了训练集），但是测试集上的loss很大。

这就是所谓的过拟合，overfitting。

怎么解决overfitting的问题呢？

  1. 第一个方向，往往也是最有效的方向，增加你的训练集

    在我们自己做应用的时候，假如说是真的出现了overfitting的问题，最简单的方法往往就是增加你的训练集

    所以今天如果训练集蓝色的点变多了，那虽然model的flexibility大，但是因为这边的点真的挺多的，用脑子简单想想，点越多这个函数的曲线也就描绘的越清晰，也就是将形状限制住了，产生了类似的曲线，但是在课程作业中不可以使用这个方法。

    那么可以怎么做呢？

    可以做数据增强，data augmentation， 这个方法并不算是使用了额外的资料的。

    Data augmentation就是，我们用一些你对于这个问题的理解，自己创造出新的资料。

    但是在做数据增强的过程中，也不可以随便做，这个augment要augment得有道理，举例来说在影像辨识里面，我们不可能说是把一个人颠倒过来再让机器学习的，因为在现实生活中不存在一个倒过来的人。

    如果让机器强行学习这种资料的话，它就有可能学到许多奇奇怪怪的东西，所以data的augmentation要根据你对资料的特性，对你现在要处理的问题的理解，来选择合适的data augmentation 的方式。

  2. 另一个解决办法就是，让你的模型不要有那么大的弹性，给点限制。

举例来说，假设我们直接限制说，现在我们的model，我们somehow猜测出，了解到x和y背后的关系就是一条二次曲线，只是我们不明确的知道这二次曲线里面每一个参数具体长什么样子

那么我们已经知道说，模型就是二次曲线（based on domain knowledge），因为二次曲线就已经有了比较大的限制，所以我们有给定了的三个点，但是因为我们能选择的function有限，你可能就会正好选到跟真正的distribution，比较接近的function，然后在测试集上得到比较好的效果。

那么有哪些方法可以给model限制呢？

这里列出了四点

  • 给它比较少的参数，如果是deep learning的话，就给它比较少的神经元的数目，本来每层一千个神经元，改成一百个神经元之类，或者可以让 model共用参数，可以让一些参数有一样的数值，CNN的部分会讲这里共用参数的问题，在前面所讲的network的架构叫做fully-connected network（就是一个全连接层），这个架构是一个比较有弹性的架构了，而后面讲的卷积神经网络CNN其实是一个有限制性的架构，那么CNN之所以厉害是因为这个限制主要针对了影像的特性，在虽然有限制但是在影像上反而会更加优秀
  • 另外一个就是用比较少的features，比如本来给三天的资料去预测，改成两天的结果，就好了一些
  • 还有可以做Early Stopping，Early Stopping，Regularization跟Dropout，都是之后会说明的。

但是我们也是不能给模型太多的限制的，给了太多的限制也是得不到好的结果的。

比如说我们的模型，一定是Linear Model，一定是写成y = a + bx，那么你的model它能够产生的function，就一定是一条直线

但是这信息给出的点明显不是线性关系，但由于限制太多怎么train也出不来一个好的结果，这里为了防止overfitting加了过多的限制之后，又反而产生了之前提到的Model bias的问题。

矛盾出现

今天你让你的模型的复杂程度，或者说是模型的弹性越来越大，但是什么叫做复杂的程度，什么叫做弹性？后面会更清晰的说明。

所谓的复杂，就是这个模型包含的function set比较大，参数多。

我们从这个图上来看，模型复杂程度来看，模型刚开始从简单到复杂的时候前半段Training loss 和 Testing loss 都是下降的，但是当我们过了一个点之后，Training loss虽然还在缓慢下降，但是Testing的loss很明显反而上升了！

这就是因为当model越来越复杂的时候，复杂到了一定的程度，overfitting的状况就会出现，所以你在training的loss上面可以得到了好的结果，但Testing loss 惨掉了。

我们当然想找到一个点，不是太复杂也不太简单，刚刚好在那个点上，我们都能得到最低的loss，怎么选呢？

其实可以想到我们假设三个模型，他们的复杂程度不同，我不知道要选择哪一个模型才会刚刚好，把这三个模型的结果都在测试集上面跑出来，然后上传到kaggle上面，你及时的知道了你的分数，看看哪个分数更低，哪个模型就是最好的模型，但是并不建议这么做。

将上面的极端例子过来，我们训练出了一堆的model ，但是都是挺垃圾的model，他们会做的是训练集里面有的资料就把他记下来，训练集没看过就直接output随机值。

那我们有好多这种类似的模型，将这些个模型不断的去计算Loss，其实就是不断的随机值，看看哪个随机的值最小，总会有一个最好的结果，但是！这样做是很危险的，随机出来的值，下一次是会变的，所以这次运气好得到了最低的loss，但是下一次随机可能会出现非常离谱的结果，心态爆炸。

为什么要把testing的set，分成public和private呢？

在kaggle上面是将testing set分为了public和private两种，想想看,假设所有的data都是public,那我刚才说,就算是一个一无是处的Model,得到了一无是处的function,它也有可能在public的data上面,得到好的结果,如果我们今天只有public的testing set,没有private的testing set,那你就回去写一个程式,不断random產生输出就好,然后不断把random的输出,上传到kaggle,然后看你什麼时候,可以random出一个好的结果,那这个作业就结束了。

这种暴力破解的方法显然不是想要的。

Cross Validation

怎么选择model才是比较合理的呢？

我们将Training的资料分为两半，一部分作为Training Set，一部分是Validation Set

有大概90%的资料去做Training Set，大概10%的资料会被拿来做Validation Set，我们在Training Set上面训练出来的模型，根据Validation Set上面的分数，去挑选结果，再把这个结果上传到Kaggle，去看看你的public的分数，那是因为你在挑分数的时候，就是用Validation Set来挑你的model，所以你的public的Testing Set就可以反应private的分数，不会说public和private的分数差距太大。

当我们看到public Testing Set上面的结果，有可能还想调整模型，但是我们要明白这个调整是不能调整太多次的，当我们调整很多次后可能fit在这个public Testing Set上面，然后在private Testing Set上面得到差的结果。

那么怎么分Training set和Validation set呢？ 可以随机分，但是我们可能会想到，随机分的话搞不好我这个分的不好，分到了很奇怪的验证集，导致结果不好。

N-fold Cross Validation

有这个担心就用，N-fold Cross Validation，这个做的就是将训练集分为N等份，在这个例子里面，我们切成了三等份，切完后我们拿其中的一份当作Validation Set，另外两份当作Training Set，重复三次。

有三个模型,你不知道哪一个是好的,你就把这三个模型,在这三个setting下,在这三个Training跟Validation的,data set上面,通通跑过一次，将这三个情况取平均，把每一个模型在这三种状况的结果都平均起来，再看看谁的结果最好。

那假设现在model 1的结果最好,你用这三个fold得出来的结果是,这个model 1最好,然后你再把model 1,用在全部的Training Set上,然后训练出来的模型,再用在Testing Set上面,好 那这个是N-fold Cross Validation。

Mismatch

它的原因跟overfitting,其实不一样,一般的overfitting,你可以用搜集更多的资料来克服,但是mismatch意思是说,你今天的训练集跟测试集,它们的分佈是不一样的，针对mismatch的问题在后面作业11有涉及。

总结

这里我们先看训练集数据，如果loss大，就先去看看是不是有model bias和优化问题，如果在training data上的loss小了再看testing data上面的loss，如果大，这说明我们overfitting或者mismatch了，调小loss后就得到了一个不错的结果。

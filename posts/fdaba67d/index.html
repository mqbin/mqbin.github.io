Regression

Machine Learning

Machine Learning 总结一句话就是让机器具备找到一个函数的能力。

其中最常见的方面分别是：

  • Regression

    回归问题我们今天找的函数的输出是一个scalar

  • Classification

    分类问题的理解就是找出的这个函式有多个选项

除了上述的方面还有Structure Learning，结构化学习，这往往是大家比较害怕触碰的问题，也就是要求机器今天不只是要做选择题，不只是输出一个scalar，还要产生一个有结构的东西。**举例来说,机器画一张图 写一篇文章,这种叫机器產生有结构的结果 **

Case1

这里举出的是李宏毅老师预测自己youtube频道访问量的例子，就是看看能不能找到一个函式，这个函式输入的就是youtube上面这个频道后台的咨询，输出的就是某一个，隔天这个频道会有的总的一个观看次数。

三大步骤

1.Function with Unkown Parameters

首先我们要写出一个，带有未知参数的函式，简单来说就是我们先猜测一下，我们打算找的这个函式F的数学式子到底长什么样子。举例来说，我们这边先做一个最初的猜测，比如先写：

y=b+w∗x1y = b + w * x_1 y=b+w∗x1​

这里我们暂时猜这个函式就是这样子。

  • yyy是我们准备要预测的东西，我们准备要预测的是2.26总观看人数
  • x1x_1x1​是这个频道，前一天总共观看的人数，yyy和x1x_1x1​都是数值
  • bbb跟www是未知的参数，它是准备要透过资料去找出来的，我们还不知道www和bbb应该是多少，这里只是写上

这里的猜测往往就来自于，个人对本问题的理解，也就是Domain Knowledge

那么这里怎么就列出来这个式子了？

我们可以这么理解，我们先把昨天的点阅次数乘上一个数值（weight），但是总不会是一模一样的，所以我们加上它的一个偏差（bias），作为这么一个预测。

基本概念

  • y=b+w∗x1y = b + w * x_1y=b+w∗x1​中x1x_1x1​是前一天的观看人次，yyy是所预测的后一天的观看人次，而www和bbb是未知的，称之为Parameter，这些未知的变量通常中文翻译为参数。

  • 这个含未知参数的Function，就称之为Model。Model这个东西在机器学习之中就是一个带有未知Parameter的Function

2.Define Loss from Training Data

我们的第二步，定义Loss 损失函数，Loss本身也是一个Function，那么这个Function的input就是我们Model里面的参数，在上面说了weight和bias都是Unkown的Parameter，这个是我们想找出来的，所谓的这个Loss，它就是一个Function的输入是Parameter，输出是可以评价这笔Parameter好坏的结果。

简单来说我们通过Loss function去评价参数优秀，是否work

如何计算Loss？

这个Loss我们一定要从训练资料中进行计算

这里我们将weight设置为1，bias设置为0.5k来看下它是否work

我们使用2017年1月1日的点阅次数去预测1月2日的点阅次数，这里算出预测值y_hat是5.3k而我们的真实值y是4.9k，如果使用e1=∣yhat−y∣e_1 = |y_{hat}- y|e1​=∣yhat​−y∣。

这里在李宏毅版的讲解中使用的y去表示预测值使用y_hat表示的真实值label

那么我们还可以使用1月2日去predict1月3日的值，…

这样依次类推我们就可以把三年每两天的误差给推出来，每一个都可以得到一个eie_iei​

接下来我们就把每一天的误差，都加起来取平均值，这样算出一个大L就是我们的Loss。

估测的值和实际的值之间的差距其实有不同的计算方法，在我们刚才的例子里，我们算是y跟y_hat之间绝对值的差距，这只是其中一种计算方法叫MAE。

MSE和MAE有非常微妙的差别，如果是做classification问题的话可能会选择Cross-entropy

这里举了一个真实的例子，根据weight和bias的不同，通过Loss画出这么一个等高线图。

这个等高线图上约偏红色系，代表计算出的Loss越大，就代表这一组w和b越差，越偏蓝色系，就代表Loss越小，就代表这一组w和b越好，这种等高线图就称之为Error Surface

3.Optimization

优化问题，我们要做的事情就是找到一个w和b，带数值进去可以让我们所求的Loss Function最小。

这里使用Optimization的方法是Gradient Descent

Gradient Descent

为了简化，我们先假设没有bbb有那个未知的参数，只有www这个未知的参数。

当我们w的值不同时，我们会得到不同的Loss值，形成了error surface，前面我们有两个参数的时候，error surface是2D的，这里只有一个参数，所以是1D的。

how to find w to make the lowest loss?

  • 首先要随机选取一个初始的点w0w_0w0​,那这个初始的点往往真的是随机的，其实也存在一些方法可以选择出一个比较好的w0w_0w0​，后面再去细说。
  • 接下来要计算，在www等于w0w_0w0​的时候www这个参数对Loss的微分是多少，∂L∂w∣w=w0\frac{\partial^{}L}{\partial w}|w = w_0∂w∂L​∣w=w0​
  • 如果左边比较高右边比较低，就把www的值变大，那么就可以让Loss缩小，如果算出的斜率是正的，就说明左边比右边低，就应该让www缩小
  • 那么这一步我们要跨多大呢？
    • 这一步的斜率有多大，斜率大自然就可以跨度大些，斜率小自然就可以小一些
    • 看learning rate（学习率）ηηη，learning rate的大小是靠自己设置的，所以这也是一种超参 hyperparameters
什么时候会停下来？
  • 第一种状况是你失去耐心了，可以在一开始就设定说，我今天调整参数，计算微分最多算几次，那具体到要算几次这也是一个hyperparameters，也是又我们决定的。
  • 还有一种是理想上的，停下来的可能是，今天我们不断调整参数，调整到一个地方，它的微分的值就是这一项，算出来正好是000，这样learning rate乘上000还是000，所以参数更新不动了。

可以发现，Gradient Descent这方法有一个很严重的问题，我们没有找到真正最好的解，我们没有找到那个，可以让Loss最小的那个www的时候就可能会停止也就是出现了局部最优解Local minima

但是其实这只是理论上的一厢情愿，在实际应用中真正的难题其实并不是local minima。

Linear Model

上面的三个步骤，就叫做训练，我们现在已经在知道答案的资料上去计算，这只是自high而已，我们只是假装不知道隔天的观看人数，然后拿这一个函数进行预测，发现误差是0.48k。

但是我们训练这个模型的目的是为了真正预测未来的观看人次。

而这里我们使用的这个模型其实就是一个简单的Linear Model。

上面我们可以看到train出来的模型在训练集上的Loss是0.48k，而在2021年测试时发现Loss变为了0.58k，所以是有误差的。

怎么做的更好？

这里可以发现，这条蓝色的线，就是红色的先平移了一下，因为我们使用隔天的观看人数去预测。但是仔细观察图，发现具有周期性，它每隔七天就会有两天下滑，发现是周五和周六下滑了，我们也可以想到周五周六要休息出去玩学习机器学习的人也就不多。

那么每隔七天一个循环，如果我们一个模型，它参考前七天的资料，把前七天的资料直接复制拿来当预测的结果，这也就要我们去修改一下我们的模型。

我们发现，修改模型之后确实改进了。这里就是使用了前七天的资料预测隔天的观看人次。

继续改进

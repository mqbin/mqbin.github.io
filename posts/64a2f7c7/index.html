Regression P2

Piecewise Linear Curves（分段线性曲线）

上次说的Linear Model其实是很简单的模型，在应用中使用Linear Model改变weight就可以更改斜率，改变bias就可以调整线的上下移动，但是无论位置，斜率如何变化，它始终都是一条线，但是我们都知道在现实中并不是这个样子的。

比如现实世界中的状态很可能是红色的那条曲线，所以仅是Linear Model是没有办法去拟合的，出现的这种问题叫Model Bias，这个Model 的Bias属于先天问题，没有办法去弥补的，除非把他的Model改掉。

注意这里的Model Bias和之前模型种的小b是两个不同的概念

有了Model Bias的问题我们应该怎么办？

我们需要一个更加complicated更加flexible的Function去拟合

这里我们说这个red curve可以由一组的blue curve去表示。

这个blue curve的特性：

  • 当输入的值，当x轴的值小于这个threshold（阈值）时，他就是某一个定值
  • 大于另一个threshold的时候，又是另外一个定值
  • 中间有一个斜坡

怎么用蓝色去表示red curve？

我们看到又图上 0+1+2+30 + 1 + 2 + 30+1+2+3加起来我们就得到了red curve

上面那个red curve是由很多线段组成的，有很多锯齿状的线段，这个就叫Piecewise Linear的Curves，那么不难发现，如果你今天 Piecewise Linear 的 Curves 越复杂,也就是这个转折的点越多啊,那你需要的这个蓝色的 Function 就越多。

我们可以使用这个Piecewise Linear Curves去逼近一个真正的曲线。

你可以用 Piecewise Linear 的 Curves,去逼近任何的连续的曲线,而每一个 Piecewise Linear 的 Curves,又都可以用一大堆蓝色的 Function 组合起来,也就是说,我只要有足够的蓝色 Function 把它加起来,我也许就可以变成任何连续的曲线

这些蓝色的function—Sigmoid Function？

如果要直接写出蓝色的function可能比较复杂，这里我们可以使用Sigmoid Function去逼进这个蓝色的Function，Sigmoid Function的数学表达式如下：

y=c11+e−(b+wx1)y = c\frac{1}{1+e^{-(b+wx_1)}} y=c1+e−(b+wx1​)1​

简单分析下这个式子，它的横轴输入是x1x_1x1​输出是yyy，输入先乘一个www再加上一个bbb,再取一个负号，取exponential，再加111作分母，前面乘上一个constant。

  • 如果x1x_1x1​的值趋近于无穷大，那e−(b+wx1)e^{-(b+wx_1)}e−(b+wx1​) 就消失了，当x1x_1x1​非常大的时候，这条边就会收敛在这个高度是ccc的地方
  • 如果今天x1x_1x1​负的非常大的时候，分母就会无穷大，constant就不会影响结果，yyy收敛在000

在表达时，通常不写exponential，直接写为：

y=c∗sigmoid(b+wx1)y = c * sigmoid(b + wx_1) y=c∗sigmoid(b+wx1​)

Hard Sigmoid

我们使用Sigmoid Function去逼近的这个蓝色的Function其实就是Hard Sigmoid。

这里我们根据需要调整b,w,cb,w,cb,w,c就可以制造出不同形状的Sigmoid Function，用各种不同形状的Sigmoid Function，去逼近这个蓝色的Fucntion。

  • 修改www会改变斜率，改变了坡度
  • 修改bbb可以将一个Sigmoid Function左右移动
  • 修改ccc就可以改变收敛的高度

red curve的表达式

公式

y=b+∑icisigmoid(bi+wix1)y = b + \sum_i {c_isigmoid(b_i+w_ix_1 )} y=b+i∑​ci​sigmoid(bi​+wi​x1​)

这里summation所做的就是将不同的蓝色的Function给加起来，最后不要忘记加上一个b，这个constant。

所以今天写出了一个这样的的Function，如果我们假设b、c、w未知，那么可以设定不同的参数去表示出不同的一个蓝色Function，将它们叠加起来就可以制造出不同的red curve，也就是Piecewise Linear Curves，有这个Curves，就可以去逼近，各式不同的Continuous Function。

我们将Linear Model改变为了一个更加灵活的Model 这样解决了前面提到的Model Bias的问题。

多个Feature

我们可以使用多个Feature，j 来代表 Feature 的编号，把本来放在这边的东西放到 Sigmoid 裡面,然后呢这个每一个 Sigmoid 的 Function 裡面呢,都有不同的 bi 不同的 wijw_i{_j}wi​j​,然后取 Sigmoid 以后乘上 ci 就全部加起来,再加上 b 就得到 y,我们只要这边 ci bi 跟 wijw_i{_j}wi​j​ 在放不同的值,就可以变成不同的 Function.

Batch

实作上，假设我们共有NNN笔资料，通常会把这NNN笔资料分为一个一个的Batch，就是一包一包东西，随机分就好。

每个Batch里面有BBB笔资料，那本来我们是把所有的 Data 拿出来算一个 Loss,那现在我们不这麼做,我们只拿一个 Batch 裡面的 Data出来算一个 Loss,我们这边把它叫 L1

也可以相像说BBB够大，也许L会和L1很接近，在实际上每次我们会先选一个 Batch,用这个 Batch 来算 L,根据这个 L1 来算 Gradient,用这个 Gradient 来更新参数,接下来再选下一个 Batch 算出 L2,根据 L2 算出 Gradient,然后再更新参数,再取下一个 Batch 算出 L3,根据 L3 算出 Gradient,再用 L3 算出来的 Gradient 来更新参数。

所以我们并不是拿L算梯度，实际上我们是拿一个Batch算出来的L1 L2 L3计算梯度，把所有的batch都看过一次，叫做一个Epoch，每一次更新参数叫做一次Update

每次更新一次参数叫做一次 Update,把所有的 Batch 都看过一遍,叫做一个 Epoch

所以一个 Epoch 的训练,你其实不知道它更新了几次参数,有可能 1000 次,也有可能 10 次,取决於它的 Batch Size 有多大

Hard Sigmoid

其实我们还可以对模型做更多的编写，Hard Sigmoid Function其实并不是很难写出来，他可以看作是两个Rectified Linear Unit的加总，Rectified Linear Unit图像如图：

它有一个水平的线，走到某一个地方有一个转折点，然后变成一个斜坡这个Function的表达式为：

c∗max(0,b+wx1)c*max(0,b+wx_1) c∗max(0,b+wx1​)

这个max(0,b+wx1)max(0,b+wx_1)max(0,b+wx1​)的意思就是，看000跟b+wx1b+wx_1b+wx1​谁比较大，比较大的那一个会被当作输出，如果b+wx1b+wx_1b+wx1​小于000，输出000，如果b+wx1b+wx_1b+wx1​大于000，输出就是b+wx1b+wx_1b+wx1​

总之这一条线，可以写成c∗max(0,b+wx1)c*max(0,b+wx_1)c∗max(0,b+wx1​)，每条不同的www不同的bbb不同的ccc，可以挪动它的位置，就可以改变这条线的斜率，这种线在机器学习中，我们叫做Rectified Linear Unit，缩写是ReLU

我们将两个ReLU叠加起来，就变成了Hard的Sigmoid

本来只需要iii个Sigmoid，我们要222个ReLU，才能合成一个Hard Sigmoid，所以这里需要用2i2i2i个ReLU，这种函数我们就叫他们Activation Function。

为什么引入非线性激活函数？

如果我们不使用激活函数，这种情况下我们每一层的输出都是上层输入的 线性函数，这点是很容易验证的，无论我们叠了多少层，输出的都是线性的组合，与只有一个隐藏层效果相当，这种情况就是多层感知机（MLP）了。

正是因为上面的原因，所以我们决定引入非线性激励函数，这样深度神经网络就有意义了。其实最早的想法是Sigmoid函数或者tanh函数，输出有界，很容易充当下一层的输入。

为什么说ReLU要比Sigmoid好点？

这里的对比种其实可以看出ReLU是要比Sigmoid要好

  1. 采用Sigmoid等函数，反向传播求误差梯度的时候，求导计算量很大，而ReLU求导是很容易的。
  2. 对于深层网络，Sigmoid函数反向传播时，很容易出现梯度消失的情况（在Sigmoid接近饱和区时，变换太缓慢，导数趋于000）,从而无法完成深层网络的训练
  3. ReLU会使得一部分神经元的输出为000，这样就造成了网络的稀疏性，并且减少了参数的相互依存关系，缓解了过拟合问题的发生。

接下来进行实验

  • 如果使用是个ReLU函数和Linear的Model对比好像并没有进步太多
  • 100个ReLU就出现了显著的差别了，100个ReLU在训练资料上由0.32k降低到了0.28k，在测试资料上也好了些
  • 使用1000个ReLU的话，1000个ReLU在训练资料上Loss更低了些，但是在没有看到过的资料上并没有太大进步。
